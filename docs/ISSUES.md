## Issue Template

When reporting issues, use this format:

### Issue #[NUMBER]: [TITLE]
- **Status:** Open / In Progress / Resolved
- **Severity:** Critical / High / Medium / Low
- **Component:** [FastAPI, EC2, SES, Docker, Jenkins, Tests]
- **Created:** YYYY-MM-DD
- **Resolved:** YYYY-MM-DD (if applicable)

**Description:**
Brief description of the issue.

**Steps to Reproduce:**
1. Step one
2. Step two
3. Step three

**Expected Behavior:**
What should happen.

**Actual Behavior:**
What actually happens.

**Error Messages:**
```
[Paste any error messages/stack traces here]
```

**Potential Solutions:**
- Solution option 1
- Solution option 2

**Notes:**
Any additional context or notes.

## Issue Tracking Legend
- **Status:** Open (not started) | In Progress (being worked on) | Resolved (fixed)
- **Severity:** Critical (blocks progress) | High (major problem) | Medium (important) | Low (nice to fix)
- **Component:** The module/system affected
